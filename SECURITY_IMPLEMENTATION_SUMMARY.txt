================================================================================
USAGE EXAMPLES FOR DEVELOPERS
================================================================================

1. VALIDATE USER INPUT:
   import { validateSecureInput } from '@/lib/security';
   
   const input = validateSecureInput(userInput, {
     minLength: 1,
     maxLength: 500,
     checkSqlInjection: true,
   });

2. CREATE SECURE SESSION COOKIE:
   import { createSessionCookie } from '@/lib/security';
   
   const cookie = createSessionCookie('session', sessionToken);
   response.headers.set('Set-Cookie', cookie);

3. IMPLEMENT RATE LIMITING:
   import { apiRateLimiter, getClientIp } from '@/lib/security';
   
   const ip = getClientIp(request);
   const limit = apiRateLimiter.check(ip);
   if (!limit.allowed) return new Response('Rate limited', { status: 429 });

4. SAFE ERROR RESPONSES:
   import { sanitizeError, logSecurityError } from '@/lib/security';
   
   try { /* code */ } catch (error) {
     logSecurityError(error, { endpoint: '/api/data' });
     return Response.json(sanitizeError(error), { status: 500 });
   }

TESTING SECURITY FEATURES
================================================================================

After deployment, verify with these commands:

1. Check security headers:
   curl -i https://yourdomain.com | grep -E "Strict-Transport|X-Frame|CSP"

2. Test HTTPS redirect:
   curl -i http://yourdomain.com
   # Should return 301/302 redirect to https://

3. Test rate limiting:
   for i in {1..105}; do curl https://yourdomain.com/api/data; done
   # Last requests should return 429 Too Many Requests

4. SSL certificate:
   openssl s_client -connect yourdomain.com:443
   # Verify: not self-signed, valid expiration
